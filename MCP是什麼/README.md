## MCP是什麼?

### 專有名詞說明:

### USER Prompt
在人工智能（AI）領域中，**`user prompt`** 指的是用戶提供給 AI 模型的輸入指令、問題或描述，用以引導模型生成特定的回應或執行某項任務。


### System Prompt
System Prompt 是指在人工智能系統中，預先設定的一段指令或描述，用以定義 AI 的角色、行為、語氣或回應方式。它作為模型的基礎上下文，指導 AI 如何處理用戶的輸入（user prompt）。通常由開發者設定，用於確保 AI 的輸出符合特定目標或規範。

作用：規範 AI 的行為，例如設定其為專業助手、幽默聊天機器人或技術顧問。
例子：你是一位專業的學術助手，以正式語氣回答問題，提供精確且結構化的回應。
與 user prompt 的區別：System prompt 是內部設定，控制整體行為；user prompt 是用戶的具體輸入，引導單次回應。

### AI Agent
**➜AI Agent** 是指一種能夠自主執行任務、與環境互動並根據輸入或目標做出決策的人工智能系統。以下是對 AI Agent 的簡短說明：

**➜定義**
AI Agent 是一個軟體實體，結合人工智能技術（如機器學習、自然語言處理或決策算法），能夠感知環境、處理信息並採取行動以達成特定目標。它通常具備自主性、適應性和互動性。

**➜主要特徵**

1. **感知能力**：AI Agent 通過感測器或輸入數據（如文字、圖像）獲取環境信息。  
2. **決策能力**：基於內建模型或規則，分析輸入並選擇最佳行動。  
3. **行動能力**：執行任務，如生成回應、控制設備或與用戶互動。  
4. **學習能力**（部分情況）：某些 AI Agent 可通過經驗學習，優化未來表現。  

**➜應用場景**  
- **聊天機器人**：如客服助手，回答用戶問題（例如本系統）。  
- **自動化系統**：如智能家居設備，控制燈光或溫度。  
- **遊戲 AI**：在遊戲中扮演角色，根據玩家行動做出反應。  
- **自主駕駛**：感知道路環境並做出駕駛決策。  

**➜與一般 AI 的區別**
AI Agent 強調自主性和與環境的動態互動，而一般 AI 可能僅專注於數據處理或單一任務（如圖像識別）。Agent 通常被設計為在特定場景中模擬「智能行為」。


### Function calls
Function Call 在 AI Agent 中是指 AI 模型調用預定義的函數或工具來執行特定任務或處理複雜請求的機制。當 AI 接收到用戶輸入（user prompt）後，若需要執行特定操作（如查詢數據庫、調用 API 或計算），它會生成一個結構化的請求，呼叫對應的函數並傳遞必要參數，然後將結果整合進回應。

作用：擴展 AI 能力，實現超出語言生成的功能，如資料檢索或外部系統互動。
例子：AI Agent 接收「查詢明天台北天氣」，通過 function call 調用天氣 API 獲取數據後回應。

### MCP
### AI Agent 使用的 MCP 是什麼？

**MCP（Model Context Protocol，模型上下文協議）** 是一種開放標準，旨在讓 AI Agent 能夠以統一的方式連接到外部工具、數據源或服務，從而執行實際任務或獲取即時信息。簡單來說，MCP 就像一個「通用插頭」，讓 AI 可以輕鬆與各種應用程式（如 Google Drive、Slack 或資料庫）互動，而無需為每種工具撰寫專屬程式碼。MCP 由 Anthropic 於 2024 年 11 月推出，目標是簡化 AI 與外部系統的整合，讓 AI Agent 更靈活、實用。

- **如何運作**：MCP 採用客戶端-伺服器架構：
  - **AI Agent（客戶端）**：AI 應用程式（如聊天機器人或程式碼助手）發送請求。
  - **MCP 伺服器**：負責將 AI 的請求轉換為具體工具的指令（例如調用 API 或存取檔案），然後將結果回傳給 AI。
  - **例子**：假設你問 AI：「幫我檢查明天日曆。」AI Agent 通過 MCP 連接到日曆應用程式的 MCP 伺服器，獲取資料後回應：「明天上午 10 點有會議。」

- **優勢**：
  - **統一標準**：不同工具使用相同的 MCP 格式，減少整合複雜性。
  - **動態發現**：AI 可以即時發現可用工具，無需預先硬編碼。
  - **雙向互動**：支持持續對話，AI 能根據工具回饋調整行動。

### MCP 與 Function Call 的差別

**Function Call（函數調用）** 是 AI 模型（如 OpenAI 的 ChatGPT）用來執行特定功能的機制。開發者預先定義一組函數（例如查天氣、計算數字），AI 根據用戶輸入選擇並調用這些函數。以下是兩者的主要區別，用簡單的方式說明：

1. **目的與範圍**：
   - **Function Call**：專注於讓 AI 執行開發者預定義的特定功能，像是「調用一個天氣 API」或「計算兩數之和」。功能範圍較窄，適合簡單任務。
   - **MCP**：提供一個標準化框架，讓 AI 動態連接到各種外部工具和數據源（如檔案系統、雲端服務），適用於更複雜、需要多步驟或即時數據的任務。

2. **整合方式**：
   - **Function Call**：需要開發者事先在程式碼中定義所有可用函數，並將其描述提供給 AI。如果新增工具，必須修改程式碼。
   - **MCP**：AI 可以動態發現 MCP 伺服器提供的工具，無需更改程式碼。例如，AI 可以自動找到並使用新的 Google Drive 伺服器來存取檔案。

3. **靈活性**：
   - **Function Call**：功能固定，AI 只能調用預先設定的函數，缺乏即時適應新工具的能力。
   - **MCP**：支持動態、雙向互動，AI 能根據任務需要選擇並使用新工具，甚至在多輪對話中調整策略。

4. **使用場景**：
   - **Function Call**：適合單一、明確的任務。例如，「查詢當前股價」只需要一個 API 調用。
   - **MCP**：適合需要與多個系統互動的複雜工作流。例如，AI 可能需要檢查日曆、預訂機票、發送確認郵件，這些都通過不同 MCP 伺服器完成。

5. **比喻解釋**：
   - **Function Call**：像是一個廚師（AI）只能使用廚房裡預先準備好的幾把刀（函數）。如果需要新工具，得先買回來並教廚師怎麼用。
   - **MCP**：像是一個智能廚房，廚師可以隨時從工具櫃（MCP 伺服器）拿取任何合適的工具，甚至能即時學習新工具的使用方法。

#### 總結
- **Function Call** 簡單直接，適合快速實現單一功能，但擴展性有限。
- **MCP** 更靈活、標準化，適合需要與多種外部系統互動的 AI Agent，能處理複雜任務並適應新工具。

兩者並不完全競爭，而是互補：Function Call 適用於簡單、內建功能；MCP 則為 AI Agent 提供更廣泛的外部連接能力。如果你想讓 AI 像一個真正的數位助手，與多種工具協作完成任務，MCP 是更強大的選擇。
