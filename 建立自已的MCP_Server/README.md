# 自訂MCP Server

 ## 什麼情況下會需要「自訂」大模型的 MCP server？

1. 要存取內部或私有資料
	•	大模型通常不能直接連上你的 內網資料庫、內部 API、檔案系統。
	•	如果希望 LLM 可以「查詢公司內部資料」或「讀寫私有文件」，就需要一個 MCP server，把這些資源包裝成安全的介面。

例子：
	•	連到公司內部的 PostgreSQL / MySQL
	•	查詢知識庫 (ChromaDB, Weaviate)
	•	讀取某些資料夾的 PDF / Word / CSV 檔案
	
---

2. 需要執行程式邏輯或運算
	•	有些任務大模型無法單靠推理完成，需要呼叫外部程式。
	•	這時候 MCP server 就像「外掛」一樣，幫 LLM 處理。

例子：
	•	呼叫 Python function 去跑數據分析
	•	呼叫 AI 模型（例如影像辨識模型、語音轉文字）
	•	控制 Raspberry Pi 的感測器 / 裝置

---

3. 想要標準化外部工具的整合
	•	MCP 的核心精神是 統一協議，讓不同 client (Claude、VSCode、Web UI) 都可以用相同方式叫用工具。
	•	如果你已經有很多 API 或 microservices，要給 LLM 使用時，不想每次寫一次「插件」──寫一個 MCP server 就能一次通用。

---

4. 安全性 / 權限控管
	•	大模型直接存取檔案或 API 風險很大（可能刪掉重要資料 🫠）。
	•	MCP server 可以作為「代理層」，只允許模型存取特定資源，避免模型做危險操作。

---

5. 課程 / 教學 / 實驗

你是老師 ✍️，這裡也有很實用的教學情境：
	•	學生寫一個簡單的 MCP server，讓 Claude Desktop 讀取本地資料或呼叫 Python function。
	•	透過這個過程，學生可以學到：
	•	API → MCP server 的封裝
	•	客戶端 / 伺服器的協議設計
	•	如何讓大模型「擴充」能力

---

📝 總結

我們會自訂大模型的 MCP server 的情境有：
	1.	讓模型存取私有/內部資料
	2.	提供外部運算或功能（工具調用）
	3.	統一多個外部 API 的接口
	4.	加強安全性，控管模型能做的事
	5.	教育用途，讓學生理解 LLM 工具化的流程
